# 书法单字切割系统

## 项目目标
从竖排书法页面图像中，精确切割出单个汉字，用于后续训练或识别。

## 核心流程

```
原始图像 → 内容区域裁剪 → OCR单字检测 → 分列 → 大小字分离 → 过滤书法列
    → 遗漏字符检测 → 连通域精确裁剪 → 去重 → 后处理收缩 → 保存
```

---

## 模块详解

### 1. detect_main_content_bbox() - 内容区域裁剪

**作用**：去除页面边缘空白/页眉页脚，聚焦主要内容区。

**原理**：
- 统计每列/每行的暗像素密度（灰度 < 130）
- 使用滑动窗口（默认 100px）寻找第一个/最后一个密度超过阈值的区域
- 阈值 = 图像高度 × min_density_ratio（默认 0.15）
- 向外扩展 20px margin 确保不切掉边缘笔画

**为什么做**：
- OCR 对空白区域敏感，裁剪后减少误检
- 提升 OCR 对竖排文字的检测稳定性
- 实验证明：裁剪后 OCR 检测到的有效字符更多（56 vs 78，但裁剪后噪声更少）

**参数**：
- `min_density_ratio`: 暗像素密度阈值比例，默认 0.15
- `window`: 滑动窗口大小，默认 100

---

### 2. get_ocr_char_boxes() - OCR 单字检测

**作用**：调用 RapidOCR 获取每个字符的边界框和置信度。

**原理**：
- 使用 `return_word_box=True` 获取字级别结果（非行级别）
- RapidOCR 返回结构：`RapidOCROutput.word_results` → 每行是一个 tuple，包含多个 `(text, score, box)` 三元组
- 将多边形框转为矩形框（取 min/max 坐标）
- 记录行号 (line_idx) 和字序号 (char_idx)

**为什么用 OCR 而非纯 CV**：
- 纯 CV（投影法/连通域）对粘连字、飞白字效果差
- OCR 已经内置了字符分割模型，定位更准
- 置信度分数可用于后续筛选

**返回格式**：
```python
[(x_min, x_max, y_min, y_max, text, score, line_idx, char_idx), ...]
```

---

### 3. refine_char_bbox() - 连通域精确裁剪

**作用**：以 OCR 框为中心，用连通域分析精确裁剪字符边界。

**为什么需要**：
- OCR 框通常偏大（包含背景）或偏小（切掉笔画）
- 书法字笔画不规则，需要像素级精确边界

**流程**：

#### Stage 1: 候选组件收集
1. 以 OCR 框为中心，扩展搜索区域（±40px X, ±60px Y）
2. 二值化（阈值 140）
3. **排除标点区域**：将标点框对应的二值区域置 0，避免标点连通域干扰
4. 连通域分析（8 连通）
5. 收集距离 OCR 框中心 < 80px（欧氏距离）的组件

#### Stage 2: 合并与精炼
1. 按面积降序排序，最大组件作为 seed
2. 只合并距离 seed 中心或当前合并框 < 30px 的组件
3. 添加 5px padding 确保笔画完整

#### Post-processing 1: 过大框收缩
- 如果精炼后面积 > 2× OCR 框面积，且 seed 组件大部分在 OCR 框内
- 则改用 seed 组件的紧凑框
- **为什么**：防止列末尾空白/噪点被误判为字符的一部分

#### Post-processing 2: 标点排除裁剪
- 如果精炼框与标点框 IoU > 30%，裁剪重叠部分
- **为什么**：避免字符框包含相邻标点

**关键参数**：
- `binary_threshold`: 二值化阈值，默认 140
- `padding`: 边界扩展，默认 5
- `search_margin_x/y`: 搜索扩展范围，默认 40/60
- `merge_radius`: 候选组件收集半径，默认 80
- `exclude_boxes`: 需要排除的标点框列表

### 4. classify_columns() - 分列

**作用**：按 X 中心坐标将字符分组为列。

**原理**：
- 字符按 X 中心排序
- 如果相邻字符 X 中心差 > 100px，认为是新列
- 动态计算列平均中心，避免累积误差

**为什么用 X 中心而非固定间隔**：
- 书法列间距不均匀
- 动态阈值适应不同排版

---

### 5. split_mixed_columns() - 大小字分离

**作用**：将同一列中的大字（正文）和小字（注释）拆分为独立子列。

**原理**：
- 尺寸阈值 120px（宽高均 >= 120 为大字）
- 大字和小字分别成列

**为什么**：
- 注释字和正文字大小差异大，不应混为一列
- 后续过滤时可按尺寸筛选

---

### 6. filter_calligraphy_columns() - 过滤书法列

**作用**：保留符合书法特征的列，合并相邻小字标注。

**流程**：
1. **第一遍**：筛选平均宽高 >= 阈值（默认 100/100）且字符数 >= 2 的列
2. **第二遍**：将 X 范围重叠/相邻（±50px）的小字列合并到主列
   - 小字需 >= 标注阈值（默认 50/50）才保留
3. **第三遍**：添加遗漏的主列
4. 按 X 排序并重新编号

**为什么合并小字**：
- 书法作品中常有旁注、批注紧挨正文
- 这些标注是作品的一部分，应保留

---

### 7. detect_missing_chars_in_gaps() - 遗漏字符检测

**作用**：在字符间隙和列末尾检测 OCR 漏检的字符。

**原理**：
1. **间隙检测**：
   - 遍历相邻字符，如果 Y 间距 > 阈值（80px）
   - 检查间隙区域暗像素比例（> 15%）
   - 连通域分析，筛选面积 > 300、长宽比合理的组件
   - 合并距离 < 40px 的碎片组件

2. **列末尾检测**：
   - 最后一个字符下方如果有 > 0.5× 平均字高的空间
   - 检查是否有显著暗像素区域
   - 取最大连通域作为遗漏字符

**为什么需要**：
- OCR 对淡墨、飞白字容易漏检
- 竖排文字底部字符常被截断

---

### 8. remove_overlapping_boxes() - 去重

**作用**：移除重叠的字符框，保留面积较大的框。

**原理**：
- 按面积降序排序
- 依次检查每个框与已保留框的 IoU
- IoU > 0.3 则丢弃

**为什么保留大框**：
- 大框更可能包含完整字符
- 小框通常是碎片或误检

---

### 9. 后处理收缩（segment_characters 内）

**作用**：修正面积异常大的框（通常是列末尾误检）。

**原理**：
- 按列分组，计算中位面积
- 如果某框面积 > 3× 中位面积，缩小到 √中位面积
- 保持中心不变

**为什么**：
- 列末尾遗漏检测可能将大片空白+噪点合并为一个框
- 收缩后更接近正常字尺寸

---

### 10. save_characters() - 保存

**作用**：将切割的字符保存为正方形图像。

**原理**：
- 以 max(w, h) + 2×padding 为正方形边长
- 字符居中放置，背景 255（白）
- 命名：`page{页码}_col{列号}_row{行号}.png`

## 版本演进

### v12（基础版）
- OCR 定位 + 简单连通域合并
- 以 OCR 框中心为基准，半径内合并所有连通域
- 无后处理

### v13-v15（增强版）
- 新增标点排除机制（exclude_boxes）
- 两阶段精炼：seed 优先 + 距离合并
- merge_dist 从 80 降到 30，减少噪声吞入
- 过大框收缩逻辑
- IoU 裁剪标点重叠区域
- 遗漏字符检测（间隙 + 列末尾）
- 去重机制（IoU > 0.3）

## 配置参数

```python
config = {
    "size_threshold": 120,          # 大小字分离阈值
    "min_chars_per_col": 3,         # 列最小字符数
    "min_char_width": 150,          # 书法字最小宽度
    "min_char_height": 150,         # 书法字最小高度
    "binary_threshold": 140,        # 二值化阈值
    "bbox_padding": 5,              # 边界框扩展
    "gap_threshold": 100,           # 间隙检测阈值
    "missing_char_min_area": 500,   # 遗漏字符最小面积
    "iou_threshold": 0.3,           # 去重 IoU 阈值
}
```

## 已知问题

1. **refine 可能切掉偏旁**：当 merge_dist=30 过小时，距离较远的偏旁组件可能被排除
2. **收缩逻辑激进**：3× 中位面积阈值可能误伤真正的长条形字符
3. **OCR 对竖排支持有限**：RapidOCR 无竖排参数，依赖裁剪提升效果
4. **小字标注过滤**：过小的标注（< 50px）会被丢弃
