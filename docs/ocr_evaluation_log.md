# OCR 评估实验日志

## 评分公式

```
Score = 100 × (正确文本数 / max(GT, DET)) × edge_factor × detection_f1

edge_factor = 1 - clamp(平均绝对边缘误差 / ref_size, 0, 0.3)
detection_f1 = 2 × precision × recall / (precision + recall)
precision = 匹配数 / 检测数
recall = 匹配数 / GT数
```

边缘误差：分别测量上下左右四边。Hungarian 算法做 GT 与检测框的位置匹配（中心距离 ≤ 60px）。

## Iter 1 (v1) — ❌ 方法缺陷

**修改：** search_margin_x 40→60, search_margin_y 100→120, merge_radius 100→120,
expand_padding 15→20, gap_threshold 100→80, missing_char_min_area 500→600,
列尾搜索 2×→2.5×, CC 合并 80→100

**结果：** evaluator 方法缺陷——`orig_idx` 在参数变化后无效。
- **教训：** GT 必须用静态基准线（baseline + corrected.json），用 Hungarian 位置匹配

## Iter 1 (v2) — ❌ 轻微退步 (91.36)

**同上参数 + 修复后的 evaluator**

| 指标 | Baseline | Iter 1 v2 |

| 均值边缘误差 | 2.1 | 2.3 |

## Iter 2 — ❌ 大幅退步 (90.34)

**修改：** binary_threshold 140→150, bbox_padding 5→3, 新增 `_is_valid_chinese`

| 指标 | Baseline | Iter 2 |
|------|----------|--------|
| Score | **91.57** | **90.34** |
| 均值边缘误差 | 2.1 | 4.0 |
| 文字准确率 | 94.0% | 94.0% |

**分析：** threshold=150 断了笔画连通行。左边缘右移 +2.6（原 +0.2），框整体右偏。
**教训：** 140 是 handwriting 的近最优值，改高会破坏连通性。

---

## Iter 3 — ❌ 过修正 (91.35)

**修改：** merge_radius 100→80, bbox_padding 5→3

| 指标 | Baseline | Iter 3 |
|------|----------|--------|
| Score | **91.57** | **91.35** |
| 均值边缘误差 | 2.1 | 3.4 |
| 文字准确率 | 94.0% | 94.0% |

**分析：** merge_radius=80 改善了右边缘（+4.2→+2.1），但丢了左边的 CC 组件（l +0.2→+2.1）。
**教训：** 100 是最优值。右偏不是 CC 抓了远处组件造成的。

## Iter 4 — ❌ 对称减 padding 伤左缘 (91.13)

**修改：** bbox_padding 5→4

| 指标 | Baseline | Iter 4 |
|------|----------|--------|
| Score | **91.57** | **91.13** |
| 均值边缘误差 | 2.1 | 2.9 |
| 文字准确率 | 94.0% (1234/1313) | **94.1%** (1235/1313) |
| 文字错误 | 79 | **78** |

**分析：** padding 5→4 给了精确的 -1px 对称偏移（l+0.2→+1.2, t-2.2→-1.2, r+4.2→+3.2, b+1.1→+0.1），但绝对值因左缘从 0.2 变成 1.2 而上升。文字准确率提升来自 `_is_valid_chinese`。

**关键认识：** padding 对称但偏差不对称。减 padding 改善了右/上/下但伤了完美的左。

## Iter 5 — ❌ 非对称校正引入方差 (91.07)

**修改：** bbox_padding 5（还原）, 新增 refine_shrink_right=3, refine_shift_top=1

| 指标 | Baseline | Iter 5 |
|------|----------|--------|
| Score | **91.57** | **91.07** |
| 符号偏差 l/t/r/b | +0.2/-2.2/+4.2/+1.1 | +0.2/-1.2/+1.2/+2.1 |
| 绝对偏差 l/t/r/b | 0.3/2.6/4.2/1.3 | **0.3**/3.5/**6.3**/2.3 |
| 文字准确率 | 94.0% | **94.1%** |
| 文字错误 | 79 | **77** |

**分析：** 符号偏差大幅改善（r+4.2→+1.2, t-2.2→-1.2），但绝对值不降反升（r 4.2→6.3）。说明校正在页间不一致——有些页过校。shift_top 拖动了整个框下移，底边也跟着下移（b+1.1→+2.1）。

**教训：** 全局固定值的非对称校正不可行，页间偏差方差太大。

## 最终结论

**评分轨迹：**
- Baseline: **91.57**（当前最优）
- Iter 1: 91.36 (−0.21)
- Iter 2: 90.34 (−1.23)
- Iter 3: 91.35 (−0.22)
- Iter 4: 91.13 (−0.44)
- Iter 5: 91.07 (−0.50)
- 跳过 CC: 73.53
- 行级等分: 64.97
- 等比外扩: 61.82

**唯一有效提升：** `_is_valid_chinese` 将文字准确率 94.0%→94.1%（79→77 个错误）。

**核心结论：**
1. **CC refine 是必需步骤**——它将漏检从 39 降到 1，边缘误差从 25px 降到 2.1px
2. **CC refine 的右偏是二阶问题**——+4.2px 的偏差在 25px 的原始误差面前微不足道
3. **所有替代 CC 的方案都大幅退步**——没有 CV 辅助的纯 OCR 精度不可接受
4. **下一步应该聚焦文字纠错（77 个错误）**，而非边缘精度
