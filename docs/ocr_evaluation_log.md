# OCR Evaluation Experiment Log

## Methodology

### Score Formula
```
Score = 100 × (correct_text / max(GT, DET)) × edge_factor × detection_f1

edge_factor = 1 - clamp(mean_abs_edge_error / ref_size, 0, 0.3)
detection_f1 = 2 × precision × recall / (precision + recall)

precision = n_matched / n_det
recall = n_matched / n_gt
```

Edge error measured on all 4 sides (l, t, r, b) independently.
Hungarian algorithm matches GT and detection boxes by center distance (max 60px threshold).

### Ground Truth Construction
- **Baseline**: `_ocr_results.json` generated by the initial (reverted) pipeline params
- **Corrections**: `_corrected.json` applied to baseline by `orig_idx`
- **Reviewed filter**: only pages with `_reviewed.json` included (28 pages)
- For each iteration: compare NEW detections against this static GT via position matching

### Test Pages
28 pages with reviewed status: 24–35, 37–44, 50, 60, 79–81, 85, 99, 100

## Iteration 1 (v1) — 2026-06-17 — ❌ Method Broken

**Objective:** Broad systematic improvement targeting right edge error (4.2px), top edge error (2.6px), and text errors.

**Changes:**
| Param | Before | After | Rationale |
|-------|--------|-------|-----------|
| search_margin_x | 40 | 60 | Capture rightward stroke components |
| search_margin_y | 100 | 120 | Slightly more vertical room |
| merge_radius | 100 | 120 | Allow farther CC components to merge |
| expand_padding | 15 | 20 | More context for re-OCR |
| gap_threshold | 100 (effective) | 80 | Better inter-char gap detection (枉 fix) |
| missing_char_min_area | 500 | 600 | Reduce false positive candidates |
| column tail search | 2× | 2.5× | More room for column-bottom chars |
| CC merge distance | 80 | 100 | Merge nearby gap candidates better |

**Result:** ❌ Evaluator broken — `orig_idx`-based GT construction fails when pipeline output changes (box count/order shift). Measured regression is artifact.
- **Lesson:** GT must be static reference from baseline + corrected.json. Use Hungarian position matching.
- **Action:** Reverted all params. Created `_ocr_results_baseline.json`. Fixed evaluator.

## Iteration 1 (v2) — 2026-06-17 — ❌ Slight Regression

**Same changes as v1** but with fixed evaluator (static GT + position matching).

**Results vs Baseline:**
| Metric | Baseline | Iter 1 v2 | Δ |
|--------|----------|-----------|---|
| Score | **91.57** | **91.36** | **-0.21** |
| Edge l/t/r/b abs | 0.3/2.6/4.2/1.3 | 0.3/2.6/4.3/1.7 | r+0.1, b+0.4 |
| Edge signed | l+0.2/t-2.2/r+4.2/b+1.1 | — | — |
| Mean edge err | 2.1 | 2.3 | +0.2 |
| Text accuracy | 94.0% | 93.9% | -0.1% |
| Missed/Extra | 1/10 | 1/10 | same |
| Detection F1 | 0.996 | 0.996 | same |

**Analysis:** Wider search margins and larger merge radius caused boxes to capture MORE noise, increasing edge error. The CC refinement already overextends rightward; enlarging the search window makes it worse.

**Lesson learned:** For CC refinement, expanding search area doesn't fix bias — it just captures more of whatever the bias already captures. Need to: (1) tighten, not widen; (2) understand signed edge bias before adjusting.

**Signed edge bias revealed:**
- `dt = -2.2px`: detected top is ABOVE GT (CC finds ascender ink)
- `dr = +4.2px`: detected right is RIGHT of GT (CC overextends rightward)
- `db = +1.1px`: detected bottom is BELOW GT (minor)
- `dl = +0.2px`: left is essentially perfect

**Conclusion:** CC refinement makes boxes slightly larger than human GT in all directions, especially rightward.

## Iteration 2 — In Progress

**Hypothesis from Iter 1:** CC overextends rightward because binary_threshold (140) includes too much ink, connecting the character to adjacent pixels. Higher threshold = less ink = tighter boxes.

**Changes:**
| Param | Before | After | Rationale |
|-------|--------|-------|-----------|
| binary_threshold | 140 | 150 | Less ink→tighter CC boxes→reduce right bias |
| bbox_padding | 5 | 3 | Reduce all edges by 2px |
| text validation | — | `_is_valid_chinese` | Filter non-CJK noise from `original_text` preference |

**Expected:** Tighter boxes reduce right edge error (4.2→3.x), padding reduction helps all edges equally. Text validation should fix ~16 errors (non-CJK chars at high confidence).

**Status:** Re-running pipeline on 28 pages.
