# 项目进展

## 项目概述

## 算法迭代记录

### v1-v12: 探索阶段

| 版本 | 方案 | 问题 |
|------|------|------|
| v1-v5 | 简单连通域分析 | 网格线干扰严重，字符被分割 |
| v6-v8 | 投影法分割 | 对不规则布局适应性差 |
| v9-v12 | OCR 直接框选 | 注释和标点被框入，所有 box 偏上不准确 |

### v13: OCR定位 + 连通域精确裁剪 + 注释清理

**方案**: OCR 检测单字 → 列聚类 → 连通域精确裁剪 → 注释清理

**参数**:
- OCR 引擎：RapidOCR (PP-OCRv4)
- 列聚类阈值：x 中心间距 > 100px
- 大字尺寸阈值：宽度/高度 >= 140px
- 连通域裁剪：OCR 框中心 100px 范围内合并
- 注释清理：面积比 >= 0.15 且 (距离 < 0.6倍主字符尺寸 或 非圆形)

**结果 (第24页)**:
- OCR 检测单字数：150 个
- 检测到列数：12 列 → 拆分为 13 个子列
- 书法列数：4 列（3 列正文 + 1 列"第一回"）
- 提取字符数：37 个

**问题**:
- 侧边标题 "硬笔行书红楼梦" 被误检为书法列
- 部分字符框不完整（如：乱、唱、罢等笔画较多的字）
- 括号等标点符号仍被识别为字符

### v14: 收紧搜索半径

**改进**: 缩小 `refine_char_bbox` 搜索半径到 ±30px Y / ±15px X

**问题**:
- 搜索半径过小，导致更多字符框不完整
- 部分字符的笔画被截断

### v15: 内容区域裁剪 + 扩大搜索半径 + 标点过滤

**改进**:
- `detect_main_content_bbox` - 通过滑动窗口检测暗像素密度，裁剪到主要书法区域，排除侧边标题
- 扩大 `refine_char_bbox` 搜索半径到 ±60px Y / ±40px X，合并半径 80px
- 显式标点过滤集，在列分类前排除括号、逗号等

**测试验证**:

| 页码 | 验证内容 | 结果 |
|------|----------|------|
| 30 | 常规页面布局 | ✅ 正常 |
| 53 | 特殊字符 (袄、蟒) | ⚠️ 部分字符可能遗漏 |
| 187 | 含括号注释 (林黛玉) | ✅ 括号已过滤 |

**问题**:
- OCR 直接框选仍然不准确，box 偏上
- 注释小字仍可能被框入

### v16: 形态学膨胀 + 连通域分析 + 书法列过滤

**方案**:
- 二值化后使用形态学膨胀连接断裂笔画
- 连通域分析提取候选字符
- 按 x 中心坐标分列
- 过滤出大字书法列（排除小字注释）

**问题**:
- 膨胀过度导致字符粘连
- 分列逻辑错误，列数太少

### v17: OCR定位 + 形态学笔画覆盖融合

**方案**:
- OCR 框作为种子定位
- 形态学膨胀获取连通域
- 融合 OCR 框和连通域边界

**问题**:
- 分列逻辑复杂，映射错误
- 字符框重叠严重

### v18: OCR定位 + 连通域精确裁剪 + 遗漏检测 + 竖线修复

**方案**:
- 在 OCR 字符间隙中查找遗漏字符（CLAHE 增强）
- 修复字符下方延伸的竖线笔画

**问题**:
- 所有字符高度异常增加
- 字符框大小不合理

### v19: OCR定位 + 连通域精确裁剪 + 相邻边界约束 (当前)

**方案**:
- OCR 检测单字 → 列聚类 → 连通域精确裁剪
- **关键改进**: 搜索范围限制在相邻字符边界之间，防止重叠
- 保守搜索范围 (±20px X, 相邻字符间 Y)
- 合并半径 80px

**关键参数**:
- 搜索范围：±20px X / 相邻字符间 Y
- 合并半径：80px
- 连通域最小面积：30px²
- 书法列过滤：平均宽高 >= 140px

**测试结果**:

| 页码 | 书法列 | 字符数 | 重叠 | 备注 |
|------|--------|--------|------|------|
| 24 | 4列 | 36字 | ✅ 无 | "第一回"已捕获 |
| 30 | 6列 | 64字 | ✅ 无 | - |
| 53 | 4列 | 46字 | ✅ 无 | - |
| 187 | 5列 | 45字 | ✅ 无 | - |

**已知问题**:
1. ❌ "枉" 字未被 OCR 识别到（需要遗漏检测补充）
2. ❌ "年" 字竖线未完整截到（搜索范围限制）
