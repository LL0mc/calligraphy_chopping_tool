# Fix: 空页预览区裂痕 — 添加默认空状态

## 问题
页面无书法内容时，预览区显示二值化处理后的裂痕/噪点，看起来像 bug。

## 根因
- Canvas `dc()` 在 `bx.length === 0` 时直接 clearRect + return，无任何视觉反馈
- 页面图片 (`_processed.png`) 在空白页上产生二值化噪点
- 无空状态 UI 处理

## 修改方案

### 1. Canvas 空状态绘制 (`review_server.py:662-666`)
在 `dc()` 函数中，当 `bx.length === 0` 时，不再直接 return，而是绘制：
- 半透明遮罩（rgba 白色 0.85 透明度）覆盖二值化噪点
- 居中提示文字："本页无识别内容"
- 轻微虚线边框标识画布区域

### 2. CSS `.img-wrap` 背景色 (`review_server.py:210-211`)
给 `.img-wrap` 添加 `background` 属性，确保图片加载前/有噪点时有干净底色。

## 修改文件
- `review_server.py` 行 210-211（CSS）
- `review_server.py` 行 662-666（JS `dc()` 函数）

## 验证
1. `python review_server.py` → 打开无内容页面 → 预览区显示"本页无识别内容"
2. 打开有内容页面 → box 标注正常显示，无影响
