# 《吴玉生红楼梦诗词字帖》单字提取与检索项目

## 项目目标

将 PDF 字帖中的每个汉字切割为独立图片，通过 OCR 识别文字，存入 Obsidian 可检索仓库，实现按汉字、诗词出处等多维度检索。

### 4.3 单字切割模块 (`char_segmenter.py`)

**核心算法：连通域分析 + 启发式过滤**

1. 二值化（阈值 ~140，经实测最优）
2. 连通域分析（`cv2.connectedComponentsWithStats`）
3. 过滤条件：
   - 面积范围：200 ~ 30000 像素
   - 宽高范围：15 ~ 200 像素
   - 宽高比：0.3 ~ 3.0（排除过长/过扁的噪声）
4. 按位置排序（从上到下，从右到左——符合竖排阅读习惯）
5. 对粘连字符：使用投影法二次分割

```python
def segment_characters(image, min_area=200, max_area=30000):
    """从页面图片中切割出所有单字，返回 [(bbox, char_image), ...]"""
```

**粘连字符处理策略：**
- 水平/垂直投影找谷底
- 在谷底位置切割
- 递归处理直到所有子区域符合字符尺寸

### 4.4 OCR 识别模块 (`char_ocr.py`)

- 使用 `CnOcr`（已安装 `ch_PP-OCRv5` 模型）或 `RapidOCR`
- 对每个切割出的单字进行识别
- 返回识别结果 + 置信度

**置信度处理：**
- 置信度 > 0.8：直接采用
- 0.5 ~ 0.8：标记为待人工校验
- < 0.5：标记为识别失败，保留图片供人工标注

## 六、关键技术点

### 6.1 切割精度保障

| 问题 | 解决方案 |
|------|----------|
| 字符粘连 | 投影法分割 + 形态学操作 |
| 标点符号混入 | 面积/宽高比过滤 |
| 网格线干扰 | 形态学开运算去除细线 |
| 笔画断裂 | 适当膨胀后连通域分析 |

### 6.2 OCR 准确率提升

| 策略 | 说明 |
|------|------|
| 字符居中放大 | 切割后将字符居中放到 128×128 白底图上 |
| 多模型投票 | CnOcr + RapidOCR 双模型识别，取一致结果 |
| 上下文校正 | 利用诗词原文校验 OCR 结果 |
| 人工校验 | 低置信度结果导出为待审列表 |

### 6.3 Obsidian 检索能力

| 检索维度 | 实现方式 |
|----------|----------|
| 按汉字搜索 | Obsidian 全局搜索 + Dataview |
| 按诗词搜索 | 标签 + 文件夹结构 |
| 按作者搜索 | frontmatter 属性 + Dataview |
| 按置信度筛选 | Dataview 查询过滤 |
| 字频统计 | Dataview 聚合查询 |
